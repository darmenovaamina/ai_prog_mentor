# IT4IT Summary — AI Копирайтер

## S2P — Strategy to Portfolio

Локальные бренды в Казахстане тратят до 30 минут на ручной копирайтинг одной карточки товара — написание текста для Instagram, Kaspi и сайта по отдельности. При каталоге в 50+ позиций это превращается в полноценную штатную единицу.

**AI Копирайтер** — инвестиция в скорость: генерация трёх форматов за 10 секунд по одному фото. ROI считается не в деньгах, а в высвобожденном времени маркетолога.

---

## R2D — Requirements to Deploy

Применялся итеративный подход к разработке системного промпта. На первых тестах модель выдавала однородный текст без учёта канала. После анализа были введены явные требования:

- **Instagram** — эмодзи, разговорный тон, хэштеги в конце
- **Kaspi** — строгие характеристики, буллеты без эмодзи, акцент на цену/качество
- **Сайт** — SEO-абзац (80–120 слов) с призывом к действию

Каждая итерация проверялась на реальных фото товаров до достижения стабильного качества вывода.

---

## R2F — Requirements to Fulfill

Инструмент поставляется как **Web App на Streamlit** — пользователь открывает браузер и работает без установки дополнительного ПО. Единственное требование на стороне клиента — наличие OpenAI API-ключа.

Это снижает порог входа до минимума и позволяет масштабировать решение на любую команду без DevOps-экспертизы.

---

## D2C — Deploy to Consume

Операционный контроль реализован на двух уровнях:

**FinOps:** затраты отслеживаются через лимиты в OpenAI Dashboard — установлен hard limit на месячный бюджет, чтобы исключить неконтролируемые расходы при росте использования.

**Надёжность интерфейса:** трёхуровневая защита от галлюцинаций модели:
1. `response_format={"type": "json_object"}` — JSON Mode в API, модель физически не может выдать не-JSON
2. `re.sub(r"```(?:json)?", ...)` — очистка markdown-обёртки на случай edge case
3. `except json.JSONDecodeError` — понятное сообщение пользователю вместо краша

## Рефлексия

**Что сработало хорошо:**
- JSON Mode резко снизил количество невалидных ответов
- Разделение промпта по каналам (Instagram / Kaspi / сайт) дало предсказуемый формат вывода
- `detail: high` в Vision API заметно улучшил качество описания товара по фото

**Что можно улучшить:**
- Добавить кэширование результатов по хэшу изображения — сейчас каждый запрос тратит токены
- Вынести системный промпт в отдельный редактируемый UI-блок для кастомизации без кода
- Добавить выбор языка (RU / KZ / EN) прямо в интерфейсе
